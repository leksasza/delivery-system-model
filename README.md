# Delivery System Simulation

**Схема сущностей внутри модели доставки**

![Схема доставки](https://i.ibb.co/ccfdJp2k/image.png "Схема")

**Описание**  
Проект демонстрирует объектно-ориентированный подход к моделированию бизнес-процессов. Сущности системы и связи между ними соответствуют следующей схеме:

- Поставщик привозит товары и пополняет склад.
- Кладовщик и курьер работают на складе: кладовщик собирает заказ, курьер доставляет его пользователю.
- Пользователь делает заказ, который состоит из товаров, хранящихся на складе.

При запуске система случайным образом генерирует поставщиков, товары и склады, после чего предлагает пользователю оформить заказ в интерактивном режиме через консоль.

**Структура проекта**  
Весь код находится в одном файле laba.py. Основные классы:

*Interface* — центральный реестр, хранящий данные обо всех объектах системы (склады, товары, поставщики) и предоставляющий методы для их взаимодействия.  
*Store* — склад. Хранит списки курьеров и кладовщиков, принимает заказы от пользователей, отправляет запросы поставщикам на пополнение товаров.  
*Provider* — поставщик. Обрабатывает запросы складов и обновляет их запасы.  
*Worker* — абстрактный базовый класс для работников. Реализует логику смен и расчёта заработной платы.  
*Courier(Worker)* — курьер. Получает собранный заказ от кладовщика и доставляет его пользователю.  
*Storekeeper(Worker)* — кладовщик. Собирает заказ из товаров на складе и передаёт его курьеру.  
*User* — пользователь. Оформляет заказ, указывая координаты и список желаемых товаров.  
*Order* — заказ. Хранит статус доставки, список товаров, время создания и завершения, идентификаторы ответственных работников.  
*Item, Item_Store, Provider_Item, Provider_Store* — вспомогательные датаклассы для хранения информации о товарах и связях между сущностями (отношения M:N реализованы через отдельные классы-связки).  

**Ключевые механизмы**  
Декоратор @froze_it запрещает добавление новых атрибутов к экземплярам после инициализации, что обеспечивает целостность данных основных сущностей (Store, User, Worker).
Выбор ближайшего склада выполняется на основе координат пользователя и текущего времени — система учитывает часы работы склада.
Время сборки и доставки заказа симулируется через time.sleep, что отражает реальную асинхронную природу процессов.
Если запрошенный товар есть в наличии, но в меньшем количестве, чем нужно, система предлагает пользователю согласиться на доступное количество или отменить позицию.
Запуск
Для работы требуется Python 3.7+. Сторонние зависимости отсутствуют.
bashpython laba.py
После запуска система инициализирует данные, сделает тестовый запрос поставщику от случайного склада, после чего предложит пользователю ввести координаты и сформировать заказ.
